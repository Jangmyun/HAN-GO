`HAN-GO_PRD_v0.3.md` §11 확정 결정사항을 기준으로 아래 구현 범위를 검토하라.

## 검토 대상

$ARGUMENTS (비어 있으면 최근 변경 파일 전체를 대상으로 한다)

## 확정 결정사항 체크리스트 (§11)

| 항목 | 확인 포인트 |
|------|------------|
| **A. 인증** | 카카오 OAuth + Guest(전화번호) 두 경로가 모두 동일한 주문 플로우를 타는지 확인 |
| **B. 결제 (B-1)** | 결제 금액 계산이 `calculate_order_total()` 단일 진입점을 통하는지 확인 |
| **D. 공연 좌석** | 좌석 선점 시 DB 행 락 + 5분 홀드 TTL이 적용됐는지 확인 |
| **E. 취소** | `preparing` 이후 취소는 반드시 스토어 승인을 거치는지 확인 |
| **F. 장바구니** | 다른 스토어 상품을 담을 때 초기화 확인 모달을 띄우는지 확인 |
| **G. 재고** | `tracked` / `manual_sold_out` 듀얼 모드가 구현됐는지 확인 |
| **J. 수수료** | 수수료 로직이 `calculate_order_total()` 외부에 하드코딩되지 않았는지 확인 |
| **L. QR (L-2)** | `qr_token` 이 서버에서만 생성·검증되고 클라이언트에서 유효성을 판단하지 않는지 확인 |

## 미확정 항목 주의 (§13)

- **J (수수료)**: 아직 미확정. 수수료 관련 로직을 구현하지 말 것.
- **K-sub (OTP)**: MVP는 단순 입력(K-sub-1). SMS OTP를 추가하지 말 것.

## 보안 포인트 (§7.2)

- `qr_token` 이 nanoid 22자 이상 랜덤값인지 확인한다.
- Guest 전화번호가 해시 또는 최소 보관 방식으로 저장되는지 확인한다.
- 결제 관련 민감 정보(카드번호·계좌 비밀번호)가 저장·처리되지 않는지 확인한다.
